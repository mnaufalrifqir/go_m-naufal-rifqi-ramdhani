# (6) Version Control and Branch Management (Git)
- Pada pertemuan ini belajar tentang apa itu versioning dan bagaimana cara menggunakan git, versioning merupakan cara kita untuk mengatur versi dari source code program, kita juga dapat melihat setiap perubahan dari code sebelumnya ke code yang baru, dan kita juga dapat mentracking setiap progres dari code yang kita buat dari awal hingga saat ini.
- Terdapat 3 tools, diantaranya : 
    1. Version control system (VCS)
    2. Source code manager (SCM)
    3. Revision control system (RCS)
- Ada 3 macam version control system, diantaranya : 
    1. Single user (lokal saja)
    2. Centralized (sistem terpusat)
    3. Distributed (lokal dan server ada)
- Salah satu contoh dari distributed yaitu Git yang akan dipelajari lebih dalam pada pertemuan ini.
- Fase-fase pada git ada 3 tahap, yang pertama adalah working directory, dari working directory jika kita melakukan git add maka akan masuk ke fase yang kedua yaitu staging area, lalu jika kita melakukan commit maka akan pindah ke fase yang ketiga yaitu repository yang tersimpan di git.
- Banyak fitur git yang bisa digunakan, seperti add, push, pull, commit, dll.
- Kita juga belajar tentang branching agar kolabolari dapat berjalan dengan optimal.
